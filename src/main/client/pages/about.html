About the tracker

The tracker is a simple web application that enables you to collaborate on a secure spreadsheet/database-like data set. Although it has some similarity to Google Docs, it is designed more to accommodate tracking events in a research study. Some of its more distinctive features include:

  • Support for multiple studies
  • Multiple views within a single study, each showing different sets of attributes
  • Attribute types include: dates, booleans, strings, and dropdowns
  • Support for N/A values
  • Fine-grained permissions on views and studies
  • Full audit logging of all value changes within a study
  • Export to Excel

Release notes

Version 2.0 — July 2nd 2015
  • Ported to Java-based build process and deployment
  • Authentication and authorization through Apache Shiro
Version 1.2 — April 15th 2015
  • Changed grid to use Handsontable, which is much closer to Excel in style
  • Attributes can be "pinned" for scrolling horizontally
  • Rows can wrap over several lines of text
  • Attribute columns can have a proportionate width set

Known issues:

  • There appears to be a memory leak when switching to and from the grid view. Performance will eventually degrade. To overcome this, refresh the page: this will start from a clean state.
  • Scrolling behaviour on Firefox seems dirty, especially when using the scroll bar. Chrome is probably cleaner.
